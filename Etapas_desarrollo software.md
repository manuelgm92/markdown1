# Etapas del Desarrollo de Software

## 1. Análisis de requisitos
Es la fase en la que se identifican, recopilan y documentan las necesidades del sistema a desarrollar. Se busca entender qué debe hacer el software y bajo qué condiciones.

![Análisis de requisitos](https://i.ytimg.com/vi/Hmtriz7Af20/maxresdefault.jpg)

### 1.1 Requisitos funcionales
Definen las funciones específicas que el sistema debe realizar (ej: registrar usuarios, generar informes).

![Requisitos funcionales](https://aiatic.solutions/wp-content/uploads/2023/03/programmers-working-on-project-in-software-development-redimensionado.png)

### 1.2 Requisitos no funcionales
Describen características de calidad del sistema como rendimiento, seguridad, usabilidad o escalabilidad.

![Requisitos no funcionales](https://aiatic.solutions/wp-content/uploads/2023/08/concepto-base-datos-linea-sistema-gestion-documentos-gestionar-archivos-forma-eficaz-lista-comprobacion-ordenador-portatil.jpg)

---

## 2. Diseño
En esta etapa se define la arquitectura del sistema, los componentes, interfaces y estructuras de datos. Sirve como plano antes de programar.

![Diseño de software](https://www.varadero.es/wp-content/uploads/2020/03/servcios-it-desarrollo-software.png)

---

## 3. Codificación y compilación
Se traduce el diseño a código fuente utilizando lenguajes de programación. La compilación transforma ese código en un programa ejecutable.

![Codificación](https://www.muycomputer.com/wp-content/uploads/2019/01/lenguaje-de-programaci%C3%B3n.jpg)

---

## 4. Pruebas
Se verifica que el software funcione correctamente y cumpla los requisitos definidos.

![Pruebas de software](https://www.godaddy.com/resources/latam/wp-content/uploads/sites/4/2024/11/portada_prueba-de-desarrollo-de-software_11zon-1.png?size=3840x0)

### 4.1 Pruebas unitarias
Evalúan componentes individuales del sistema de forma aislada.

![Pruebas unitarias](https://www.inesdi.com/sites/default/files/inline-images/xprueba,P20unitaria,P20-,P201.jpg.pagespeed.ic.eOVpN9YyR-.jpg)

### 4.2 Pruebas de integración
Comprueban que los distintos módulos del sistema funcionen correctamente al interactuar entre sí.

![Pruebas de integración](https://www.valido.ai/wp-content/uploads/2024/02/What-is-Integration-Testing-A-Beginners-Guide-For-Ecommerce-1024x538.png)

---

## 5. Explotación
El sistema se pone en producción y es utilizado por los usuarios finales.

![Explotación](https://www.questionpro.com/blog/wp-content/uploads/2024/05/2722-Portada-software-para-el-recorrido-del-cliente.jpg)

---

## 6. Mantenimiento
Incluye corrección de errores, mejoras y adaptación del software a nuevos requisitos o entornos.

![Mantenimiento](https://tigroup.cl/wp-content/uploads/2024/01/computer-troubleshooting-concept-illustration_114360-7616-1.webp)

---

## 7. Documentación
Consiste en la creación de documentos que describen el sistema y su uso.

![Documentación](https://www.cesarcancino.com/wp-content/uploads/2025/04/image-1-1500x999-1.jpeg)

### Tipos de documentos
- Documentación técnica
- Manual de usuario
- Documentación de requisitos
- Documentación de diseño
- Documentación de pruebas
